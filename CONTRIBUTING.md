# Contributing

Thanks for your interest. This plugin is small and opinionated, so most changes are either **fixing a bug in a template**, **adding or updating a checklist rule**, or **improving the refresh script**. Feel free to open an issue before starting a larger change so we can agree on direction.

## Development setup

```bash
git clone https://github.com/efacsen/claude-agent-sdk-python-skill.git
cd claude-agent-sdk-python-skill
python3 -m venv .venv && source .venv/bin/activate
pip install -r scripts/requirements.txt claude-agent-sdk pytest pytest-asyncio
```

## Running the tests

```bash
pytest
```

Expect 16 tests across three files:
- `tests/test_refresh_docs.py` — exercises `scripts/refresh_docs.py` against the HTML fixture
- `tests/test_template_07.py` — exercises the dependency-injected agent in template 07
- `tests/test_templates.py` — smoke tests all seven templates via `--dry-run`

> **Note:** `tests/test_refresh_docs.py` runs the real `scripts/refresh_docs.py` against `tests/fixtures/python_docs_snapshot.html` and writes to `skills/claude-agent-sdk-python/references/`. This means running `pytest` will reset the reference files to match the fixture. If you want the references to reflect the current live docs, run `python3 scripts/refresh_docs.py` after your tests.

## Refreshing the docs snapshot

The curated Python Agent SDK references under `skills/claude-agent-sdk-python/references/` are generated by `scripts/refresh_docs.py`, which fetches the live docs, converts HTML → Markdown, splits by H2/H3 heading, and atomic-writes topic files keyed by the anchors in `sources.json`.

```bash
python3 scripts/refresh_docs.py
```

If a heading on the live docs page has been renamed, update `skills/claude-agent-sdk-python/references/sources.json` — specifically the `section_anchors` list for the affected topic file — and re-run the script.

## Running acceptance tests

Acceptance tests are manual prompts you run in a Claude Code session with the plugin installed. They live in `tests/acceptance.md` (10 tests: AT-01 through AT-10). Record results by appending a `## Results (vX.Y.Z, YYYY-MM-DD, machine)` section at the bottom.

## Commit style

- Use present-tense imperative: "Add hook signature rule", not "Added hook signature rule"
- Explain *why* in the body when the *what* isn't obvious from the diff
- One logical change per commit — easier to bisect and revert
- No bot attribution trailers (no `Co-Authored-By: Claude`)

## Adding a new template

If you add a template under `skills/claude-agent-sdk-python/templates/`:

1. Include a top-of-file docstring explaining the routing branch it addresses and cite the relevant `references/*.md` files
2. Add an `--dry-run` flag that exits 0 without hitting the network — this is what the smoke test harness in `tests/test_templates.py` exercises
3. Update `test_all_seven_templates_present` in `tests/test_templates.py` if you change the count
4. Update the routing decision tree in `skills/claude-agent-sdk-python/SKILL.md` to point at the new template

## Adding a checklist rule

Each rule in `skills/claude-agent-sdk-python/checklists/*.md` uses an `#anchor` heading (e.g. `## #no-bypass`) so the audit command can cite it by ID. When adding a rule:

1. Keep the anchor kebab-case and unique across *all* checklists (the audit command cites as `<checklist>#<anchor>`)
2. Include a short rationale explaining *why* the rule exists — a rule with only "do this" lands worse than one with "do this because X"
3. Include a good/bad code example when the rule is about syntax

## Reporting bugs

Open an issue at https://github.com/efacsen/claude-agent-sdk-python-skill/issues with:

- What you ran (`/agent-sdk:new ...`, `/agent-sdk:audit path/to/file.py`, etc.)
- What you expected
- What happened instead
- Your `claude_agent_sdk` version if relevant (`pip show claude-agent-sdk`)
